# 多模态脑肿瘤融合最小验证阶段性成果说明

## 1. 阶段目标

本阶段工作的目标不是直接追求最优分割或分类精度，而是先完成一个可运行、可解释、可展示的“最小验证”系统，验证多模态 BraTS 脑肿瘤数据上的关键技术链路能够完整打通。具体来说，重点验证以下三点：

1. BraTS 多模态 MRI 数据是否能够被稳定读取、归一化、采样并构造成训练样本。
2. 轻量级多模态融合网络是否能够将卷积特征提取、图注意力建模和扩散式特征生成联合起来完成端到端训练。
3. 模型训练结果是否能够通过可视化方式给出直观证据，支撑后续进一步扩展到更完整任务。

## 2. 当前最小验证系统的实现内容

目前代码已经围绕单病例 patch 级别分类任务搭建了一个完整闭环，主要包括以下四部分：

1. 数据侧：从 BraTS 病例中读取 T1、T2、FLAIR、T1ce 和分割标注，对非零区域做百分位归一化，再从肿瘤区域附近采样二维 patch，构造三分类标签，分别对应背景、肿瘤相关区域（水肿/坏死）和增强肿瘤。
2. 编码侧：针对 T1、FLAIR、T1ce 三个模态分别使用轻量 CNN 编码器提取 patch 特征。
3. 融合侧：先使用两层 GAT 建模不同模态之间的关系，再引入条件扩散模块，用其余模态特征去预测目标模态的噪声，从而学习模态间的互补生成关系。
4. 输出侧：训练结束后自动生成切片展示图、训练结果图、扩散过程图和图拓扑图，形成较完整的实验展示材料。

## 3. 最小验证数据与运行结果

本阶段使用了两个 BraTS 示例病例进行验证：

1. `BraTS2021_00002`
2. `BraTS2021_00060`

两例数据的体数据尺寸均为 `240 × 240 × 155`，说明当前流程已经能够稳定处理标准 BraTS 三维 MRI 数据。进一步统计如下：

| 病例 | 肿瘤非零体素 | 有效切片数 | patch 数 | patch 标签分布 |
|---|---:|---:|---:|---|
| `00002` | 190,594 | 85 | 200 | 背景 78，水肿/坏死 117，增强 5 |
| `00060` | 240,373 | 79 | 200 | 背景 88，水肿/坏死 85，增强 27 |

可以看出，两例数据都成功完成了肿瘤区域切片筛选和 patch 采样，但类别分布并不完全均衡，尤其 `00002` 的增强肿瘤 patch 数量较少。这也说明当前最小验证更适合证明“流程可跑通”，而不是作为严格的性能评测设置。

模型方面，当前网络总参数量约为 `815,235`，属于较轻量的验证型结构。它已经能够在较小样本设置下完成多损失联合训练，包括任务分类损失、扩散损失和图对比损失。

## 4. 当前结果能够证明什么

### 4.1 数据链路已经打通

从 `brats_slices.png` 可以看出，系统已经能够在 T1、T2、FLAIR、T1ce 四个模态下对同一病例生成轴状面、冠状面、矢状面的三视图，并叠加肿瘤分区标注。这说明：

1. 多模态 MRI 读取正确。
2. 不同模态之间的空间对应关系正确。
3. 分割标签与原始影像的配准关系正确。
4. 后续 patch 采样和展示所依赖的数据基础是可靠的。

### 4.2 模型已经能够稳定完成端到端训练

从 `00060` 的训练日志来看，在 60 个 epoch 内，总损失由 `1.0487` 下降到 `0.1421`，任务损失由 `0.8904` 下降到 `0.0479`，图对比损失由 `1.1841` 下降到 `0.4113`，扩散损失由 `0.9905` 下降到 `0.7360`。这表明：

1. 主任务分类分支已经能够有效拟合当前 patch 级三分类任务。
2. 图结构分支在训练过程中也在持续收敛，说明模态间关系建模确实参与了优化。
3. 扩散分支的损失下降幅度相对更缓，说明它已经开始学习模态间互补关系，但目前仍处于“可运行、可观察”的初始验证阶段。

从两例 `brats_results.png` 的损失曲线观察，整体趋势是一致的，即总损失、任务损失和图损失均随训练推进明显下降，说明该最小系统具有可重复性，而不是只在单个病例上偶然生效。

### 4.3 图学习模块已经产生了稳定输出

从 `graph_topology.png` 和 `brats_results.png` 中的 GAT 注意力热图可以看到，三种模态之间都形成了稳定的注意力分配，平均权重大多在 `0.33` 左右，且不同 head 的注意力模式存在轻微差异。这说明：

1. 图网络已经能够正常接收三模态特征并输出模态间关系矩阵。
2. 每个注意力头都能产生结构化结果，而不是数值异常或训练失稳。
3. 目前的注意力仍然较接近均匀分布，说明模型已经“会连图”，但尚未学出特别强的模态偏置关系。

因此，当前阶段可以把图学习模块界定为“功能已接入并稳定运行”，但其关系建模能力还有进一步提升空间。

### 4.4 扩散模块已经完成特征层面的最小验证

从 `diffusion_process.png` 和 `brats_results.png` 中可以看到：

1. 前向扩散过程中，特征分布随时间步逐渐向噪声分布演化，说明噪声注入过程正确。
2. 逆向去噪过程可以输出连续轨迹，说明条件去噪器能够正常工作。
3. 真实 FLAIR 特征与条件生成 FLAIR 特征在分布上已经出现一定程度重叠，说明扩散模块已经具备了“利用其他模态补充目标模态特征”的基础能力。

需要强调的是，这里验证的是“特征空间的扩散生成是否可行”，而不是像素级图像生成质量。因此目前更适合作为方法可行性的展示，而不是最终生成效果评估。

## 5. 现阶段成果的核心结论

综合代码和输出结果，本阶段已经完成了一个具有代表性的多模态融合最小验证系统，核心结论可以概括为：

1. BraTS 多模态 MRI 数据加载、预处理、肿瘤区域采样和标签构建流程已经跑通。
2. 由 CNN 编码、GAT 融合和条件扩散组成的多模态融合网络已经实现，并能完成端到端训练。
3. 两个真实病例上的训练和可视化结果表明，该方案不仅能运行，而且能产生可解释的中间结果，包括损失收敛、模态关系图和扩散轨迹。
4. 当前系统已经具备作为后续完整研究原型的基础，能够支持下一阶段向多病例、多任务和更严格评估继续扩展。

## 6. 当前边界与不足

为了在汇报中保持客观，也建议同步说明当前阶段仍属于“最小验证”，主要边界包括：

1. 训练样本规模很小，目前仅在两个病例上做了演示性验证。
2. 当前任务是 patch 级三分类，不是完整三维肿瘤分割，因此还不能直接代表临床级分割性能。
3. patch 标签由局部区域主导类别映射得到，属于一种简化标签设计。
4. 类别分布存在不均衡，尤其增强肿瘤样本偏少，会限制当前结果的泛化解释。
5. 图注意力目前尚未表现出非常鲜明的模态依赖结构，扩散模块也主要停留在特征分布层面的初步验证。

## 7. 下一阶段工作建议

在现有最小验证基础上，后续可以按以下方向推进：

1. 从单病例演示扩展到多病例训练与验证，建立更可靠的定量评估。
2. 引入训练集、验证集和测试集划分，并补充准确率、F1、AUC 或 Dice 等指标。
3. 优化 patch 采样策略，缓解类别不均衡问题，增强增强肿瘤类别的学习效果。
4. 将当前特征级融合框架进一步扩展到分割网络中，验证其对像素级肿瘤分区任务的提升。
5. 进一步分析 GAT 注意力模式与扩散生成特征的医学意义，提升模型解释性。

## 8. 总结

本阶段完成了一个面向 BraTS 多模态脑肿瘤数据的最小验证原型。系统以 T1、FLAIR、T1ce 三模态 patch 为输入，构建了“卷积编码 + 图注意力融合 + 条件扩散建模”的联合网络，并在真实病例上实现了端到端训练和全流程可视化。当前结果表明，数据读取、肿瘤区域采样、多模态关系建模、扩散式特征补全和结果展示等关键环节均已打通，模型在两个 BraTS 病例上表现出稳定的损失收敛和可解释的中间结果。整体上，这一工作已经完成了方法可行性的阶段性验证，为后续扩展到多病例、定量评估和完整分割任务奠定了基础。
